# SQL Analytics Portfolio

A portfolio project demonstrating SQL skills using a realistic Indian e-commerce
dataset built with DuckDB. Covers 20 queries across beginner, intermediate, and
advanced levels — automatically executed and published to GitHub Pages via
GitHub Actions on every push.

## Project Overview

The dataset simulates a real-world Indian e-commerce platform with 12 tables
covering customers, products, orders, payments, shipments, returns, and reviews.
Data is generated using Faker with a fixed random seed, ensuring reproducible
results on every run. DuckDB is used as the database engine — it requires no
server setup and runs entirely in-process via Python.

## Schema — 12 Tables

| Table               | Description                                      |
|---------------------|--------------------------------------------------|
| customer_segments   | Customer tier classifications                    |
| customers           | Customer profiles with location and demographics |
| categories          | Product category definitions                     |
| suppliers           | Supplier details with ratings                    |
| products            | Product catalogue with price and cost            |
| coupons             | Discount coupons with validity periods           |
| orders              | Order headers with status and total amount       |
| order_items         | Line items within each order                     |
| payments            | Payment attempts and statuses per order          |
| shipments           | Shipment tracking with carrier and status        |
| returns             | Return requests with reasons and refund amounts  |
| reviews             | Customer ratings and review text per product     |

## How to Run Locally

1. Clone the repository
git clone https://github.com/Ritwik240/SQL_analytics.git

cd sql-analytics-portfolio

2. Install dependencies
pip install -r requirements.txt

3. Generate the dataset
python scripts/generate_data.py

4. Run all queries and generate the report
python scripts/run_queries.py

5. Open the report
reports/index.html

Open this file in any browser.
